---
title: Quickstart
description: Get MyLights running locally in minutes — from a fresh clone to browsing your media library.
---

<Steps>
  <Step title="Prerequisites">
    Before you begin, make sure you have the following installed:

    - **XAMPP** (includes Apache and MySQL/MariaDB) — [download from apachefriends.org](https://www.apachefriends.org/)
    - **Node.js 18 or later** — [download from nodejs.org](https://nodejs.org/)
    - **Git**

    Clone the repository:

    ```bash
    git clone <your-repo-url> mylights
    cd mylights
    ```
  </Step>

  <Step title="Start XAMPP">
    Open the XAMPP Control Panel and start both the **Apache** and **MySQL** modules.

    Once both show a green status, Apache is serving on port 80 and MySQL is available on port 3306.

    <Tip>
      If port 3306 is already in use on your machine, XAMPP may bind MySQL to port 3307 instead. If that happens, open `backend/config/db.php` and update the port value from `3306` to `3307`.
    </Tip>
  </Step>

  <Step title="Import the database">
    Open **phpMyAdmin** at [http://localhost/phpmyadmin](http://localhost/phpmyadmin), then:

    1. Click **New** in the left sidebar and create a database named `media_host_db`.
    2. Select the `media_host_db` database, go to the **Import** tab, and import `db/main.sql`.

    This creates the full schema — `roles`, `users`, `media`, `media_type`, `category`, and `media_category`.

    <Note>
      The repository includes `db/mock_data.sql` with sample media entries. Import it after `db/main.sql` to populate the library immediately and skip manual data entry during development.
    </Note>

    Alternatively, run the import from the command line:

    <CodeGroup>

    ```bash macOS / Linux
    /Applications/XAMPP/xamppfiles/bin/mysql -u root media_host_db < db/main.sql
    # Optional mock data:
    /Applications/XAMPP/xamppfiles/bin/mysql -u root media_host_db < db/mock_data.sql
    ```

    ```powershell Windows
    & "C:\xampp\mysql\bin\mysql.exe" -u root media_host_db < db\main.sql
    # Optional mock data:
    & "C:\xampp\mysql\bin\mysql.exe" -u root media_host_db < db\mock_data.sql
    ```

    </CodeGroup>
  </Step>

  <Step title="Deploy the backend">
    Copy the `backend/` folder into XAMPP's `htdocs` directory so Apache can serve the PHP API.

    <Tabs>
      <Tab title="Windows">
        Run the provided deploy script from the project root:

        ```powershell
        .\deploy.ps1
        ```

        Or copy manually:

        ```powershell
        Copy-Item -Recurse -Force backend C:\xampp\htdocs\mylights-api
        ```
      </Tab>
      <Tab title="macOS / Linux">
        Run the provided deploy script from the project root:

        ```bash
        bash deploy.sh
        ```

        Or copy manually:

        ```bash
        cp -r backend /Applications/XAMPP/xamppfiles/htdocs/mylights-api
        ```
      </Tab>
    </Tabs>

    After copying, the API is reachable at `http://localhost/mylights-api/api`.

    <Warning>
      The frontend API client in `frontend/lib/api-client.ts` has the backend URL hardcoded to `http://localhost/backend/api`. If you deploy the backend to a different path (e.g., `mylights-api`), you must update that constant to match — otherwise all API requests will fail.
    </Warning>
  </Step>

  <Step title="Install and start the frontend">
    From the `frontend/` directory, install dependencies and start the development server:

    ```bash
    cd frontend
    npm install
    npm run dev
    ```

    The `dev` script starts Next.js on port 4000:

    ```json
    "dev": "next dev -p 4000"
    ```

    You should see output like:

    ```
    ▲ Next.js 16.1.6
    - Local: http://localhost:4000
    ```
  </Step>

  <Step title="Open MyLights">
    Navigate to [http://localhost:4000](http://localhost:4000) in your browser.

    If you imported `db/mock_data.sql`, your media library will already be populated and ready to browse.
  </Step>
</Steps>
